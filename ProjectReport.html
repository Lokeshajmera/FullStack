🏛️ About This Project

The PCCOE AI & ML Department Website is a full-stack institutional web platform built for the Department of Computer Science & Engineering (AI & ML) at Pimpri Chinchwad College of Engineering. The platform provides comprehensive departmental information, AI-powered academic tools including a PDF summarizer and AI tutor, a Teacher Portal with dashboard, student chapter showcases, research publication databases, and dynamic announcement boards — all driven by a React SPA frontend backed by a Node.js/Express REST API and MongoDB Atlas.

1.1 Statement of Work (SOW)

The objective of this project is to design, develop, and deploy a fully-featured departmental website for the AI & ML department at PCCOE. The platform targets students, faculty, and administrative staff. It delivers structured academic content, interactive student chapter pages, research & publication data, a secured Teacher Portal, and AI-powered study assistance tools — all within a responsive, accessible, and visually modern interface.

3.1 System Architecture Diagram
Presentation Layer

Node.js · Express.js · JWT Middleware · Multer (file uploads) · CORS · pdf-parse · asyncHandler · 5 Route Modules
AI Services Layer
Groq API (LLaMA 3.3-70B) → PDF Summarizer & PYQ Analyzer · Google Gemini 1.5 Flash → AI Tutor Q&A · Chunked text processing (45 000 chars)
Data Layer
MongoDB Atlas · Mongoose ODM · 4 Collections: Faculty · Student · Notice · Contact · Cloud-hosted with automated backups

Figure 1: PCCOE AI & ML Website — 4-Tier System Architecture

The platform adopts a Monolithic 4-Tier Architecture deployed on Render. The Presentation Layer is a React SPA built with Vite. The Application Logic Layer is a Node.js/Express server handling REST API calls asynchronously via HTTPS. An independent AI Services Layer proxies requests to Groq and Gemini APIs with chunking. The Data Layer is a MongoDB NoSQL database on Atlas.

4.1 AI PDF Summarizer Flow
START: Student clicks "Summarize PDF" in Study Resources
Upload PDF via multipart/form-data → POST /api/ai/summarize
Multer stores file in memory buffer (memoryStorage)
pdf-parse extracts full text from buffer
Text chunked into 45 000-char segments (chunkText utility)
Each chunk → Groq LLaMA 3.3-70B: deep technical extraction
All chunk summaries combined → final Groq prompt (JSON mode)
JSON study guide (7 sections) returned to frontend
SmartSummaryModal renders formatted study guide for student
END: Student reads exam-ready notes on screen

6.4 Installation Guide (Local Development)
Backend Setup
  • cd backend → npm install — installs all Node.js dependencies
  • Create .env file with MONGO_URI, GROQ_API_KEY, GEMINI_API_KEY
  • npm run dev — starts Express server on port 5000 with nodemon hot-reload
  • Optional: node seed.js — seeds MongoDB with sample faculty and notice data
Frontend Setup
  • cd frontend → npm install — installs React, Vite, TailwindCSS, Framer Motion
  • Configure vite.config.js proxy to point /api to http://localhost:5000
  • npm run dev — starts Vite dev server on port 5173 with HMR
  • npm run build — generates production bundle in /dist folder
