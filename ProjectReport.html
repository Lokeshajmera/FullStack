PCCOE AI & ML Department Website Phase 1 — Planning
1

Planning Phase

Statement of Work (SOW) · Resource Plan · Risk Management Plan

1.1 Statement of Work (SOW)

The objective of this project is to design, develop, and deploy a fully-featured departmental website for the AI & ML department at PCCOE. The platform targets students, faculty, and administrative staff. It delivers structured academic content, interactive student chapter pages, research & publication data, a secured Teacher Portal, and an AI-powered study assistance tool (PDF Summarizer) — all within a responsive, accessible, and visually modern interface.

PCCOE AI & ML Department Website Phase 3 — Design

Design Phase

System Architecture · DB Design · Software Components · API Docs

3.1 System Architecture

The platform employs a decoupled modern 4-tier MERN stack framework:

  • Frontend Layer: React SPA packaged via Vite. Deployed globally using Vercel.
  • Backend Layer: Node.js & Express server hosted on Render, handling REST operations.
  • AI Middleware: Streams binaries into RAM, parsing via pdf-parse, piping to Groq.
  • Data Layer: MongoDB Atlas NoSQL cluster using Mongoose ODM schemas.

PCCOE AI & ML Department Website Phase 4 — Implementation
4

Implementation Phase

Code Highlights · Technical Libraries · Code Review

4.1 Implementation Highlights
  • PDF Data Processing: Deployed a secure pipeline using pdf-parse combined with Multer's memoryStorage. Streams analysis entirely within RAM bypassing risky disk I/O.
  • Smart Chunking Logic: Algorithm to divide large documents into boundary-safe payload segments to prevent LLM rejection from Groq.
  • React Security State: Centralized JWT authorization tracking via seamless Context API wrappers.
4.2 Key Library Implementations

PCCOE AI & ML Department Website Phase 6 — Deployment
6

Deployment Phase

Release Notes · Deploy Configuration · Installation Checklist

6.1 Deployment Architecture

The Frontend activates a Vercel CI environment running npm run build, pushed dynamically out to edge-networks. The Backend activates on Render, pulling NPM packages and spinning the Express listener binding to MongoDB Atlas instances.

6.2 Master Verification Checklist
  • [x] Vercel Domain Live: Main UI interface actively responding securely on public route endpoints.
  • [x] Render API Warm: Backend responsive allowing immediate data traversal over HTTPS.
  • [x] CORS Policies Solidified: Allowances securely restricted linking Vercel explicitly to Render cleanly.
  • [x] MongoDB Permissions Validated: Mongo networks accept whitelisted 0.0.0.0 addresses ensuring server access globally.
  • [x] GROQ_API_KEY Configured: Dashboard environmental variable securely mapped operating AI processes rapidly.
  • [x] SPA Fallback Defined: Navigation rewrites natively map sub-directories smoothly preventing manual 404 URL errors safely.
